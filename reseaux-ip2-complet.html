Q QCM — Réponses & Explications

1. Quel ensemble représente une socket ?
✓ a) 192.168.1.1:80
Socket = adresse IP + numéro de port. Identifie un processus précis sur une machine précise. IP seule → machine uniquement. Port seul → service sans machine. MAC → couche liaison, pas transport.
2. Action d'un client lors d'une communication UDP ?
✓ d) Le client sélectionne au hasard un numéro de port source
UDP est sans connexion : pas de SYN, pas de 3-way handshake, pas de numéro de séquence. Le client choisit un port source éphémère aléatoire (>1023). Les options a/b/c décrivent TCP.
3. Ce qui est vrai concernant DNS ?
✓ b) Il fait la résolution des adresses IP (noms → IP)
DNS résout les noms de domaine en @IP (google.com → 142.250.x.x). Utilise UDP port 53 pour les requêtes standard. Option a = ARP (MAC↔IP). Option c = faux, DNS n'utilise pas TCP pour les requêtes normales.
4. Couche OSI responsable de l'établissement/fin de session entre applications ?
✓ c) Couche Session (couche 5)
Couche 5 = Session : établit, maintient et termine les sessions. Couche 7 (Application) = interface utilisateur. Couche 4 (Transport) = fiabilité des données, ports.
5. Affirmation vraie sur UDP ?
✓ c) Sans connexion, ne retransmet pas les données perdues
UDP = best effort. Usages : DNS, streaming, VoIP, jeux — vitesse > fiabilité. Options a/b/d décrivent TCP (numéros d'ordre, connexion 3 étapes, livraison ordonnée).
6. Champs présents dans TCP mais absents dans UDP ?
✓ b) Numéro d'ordre + d) Numéro d'accusé de réception
Port source et somme de contrôle existent dans les deux. Numéro de séquence et ACK sont exclusifs à TCP pour garantir ordre et fiabilité.
7. Protocole pour récupérer des e-mails depuis un serveur vers un client ?
✓ b) POP3
POP3 (port 110) = télécharger emails du serveur → client. SMTP (port 25) = envoyer. IMAP (143) = consulter sans télécharger. SNMP (161) = supervision réseau.
8. Rôle du numéro de port dans TCP/UDP ?
✓ b) Identifier l'application ou le service destinataire
@IP identifie la machine. Port identifie le processus/service sur cette machine. Port 80=HTTP, 443=HTTPS, 22=SSH, 53=DNS, 25=SMTP.

PARTIE I Couche Transport — TCP & UDP

🔵 TCP — Transmission Control Protocol

  • Orienté connexion (3-way handshake)
  • Fiable — accusés de réception (ACK)
  • Ordonné — numéros de séquence
  • Contrôle de flux (taille de fenêtre)
  • Retransmission si perte
  • Surcharge réseau plus importante
  • HTTP, FTP, SMTP, SSH, Telnet

🟢 UDP — User Datagram Protocol

  • Sans connexion (aucun setup)
  • Pas de garantie de livraison
  • Pas d'ordre garanti
  • Pas de contrôle de flux
  • Pas de retransmission
  • Rapide — faible surcharge
  • DNS, DHCP, VoIP, streaming

🤝 3-Way Handshake TCP

1
SYN — Client → Serveur
Client envoie SYN avec ISN = x. Demande ouverture de connexion.
2
SYN-ACK — Serveur → Client
Serveur répond SYN (ISN = y) + ACK (= x+1). Accepte et propose son numéro de séquence.
3
ACK — Client → Serveur
Client envoie ACK (= y+1). Connexion établie, transfert commence.

PARTIE II Couche Application

DNS

  • Traduit noms de domaine → adresses IP
  • Ex : google.com → 142.250.74.46
  • Port 53, protocole UDP
  • Résultats mis en cache (TTL)
  • N'utilise pas ARP (c'est MAC↔IP)

Messagerie électronique

  • SMTP (25, TCP) : envoyer des mails
  • POP3 (110, TCP) : télécharger + supprimer du serveur
  • IMAP (143, TCP) : lire sans supprimer du serveur
  • Flux : client → SMTP → serveur → POP3/IMAP → destinataire

📈 Évolution : RARP → BootP → DHCP

ProtocoleFournitLimite principale
RARP@IP seulementDiffusion couche 2 uniquement, routeurs ne propagent pas. Pas de masque ni passerelle.
BootP@IP + nom fichier boot + @serveur@IP statique par MAC dans fichier de config. Pas de bail temporaire. Utilise TFTP pour le fichier boot.

PARTIE III DHCP — Dynamic Host Configuration Protocol

📦 3 types d'allocation DHCP

Manuelle

Identique à BootP. Table MAC ↔ @IP statique. Même adresse à chaque connexion.

Automatique

@IP permanente allouée lors de la première connexion au réseau.

Dynamique ← la plus courante

@IP allouée pour une durée limitée (bail). Idéal pour utilisateurs nomades.

Ports DHCP (UDP) : Port 67 = serveur BootP/DHCP | Port 68 = client BootP/DHCP

🔄 Processus DORA — 4 messages (à connaître par cœur)

D
DHCP DISCOVER — Hôte → Broadcast
IP src: 0.0.0.0 | IP dst: 255.255.255.255
MAC src: MAC_du_PC | MAC dst: FF:FF:FF:FF:FF:FF
"Je démarre, qui peut me donner une IP ?"
O
DHCP OFFER — Serveur → Broadcast
IP src: @IP_serveur_DHCP | IP dst: 255.255.255.255
MAC src: MAC_serveur | MAC dst: MAC_du_PC
"Je te propose l'IP X.X.X.X pour Y secondes"
R
DHCP REQUEST — Hôte → Broadcast
IP src: 0.0.0.0 | IP dst: 255.255.255.255
MAC src: MAC_du_PC | MAC dst: FF:FF:FF:FF:FF:FF
"J'accepte l'offre du serveur X" (broadcast pour informer tous les serveurs DHCP)
A
DHCP ACK — Serveur → Broadcast
IP src: @IP_serveur_DHCP | IP dst: 255.255.255.255
MAC src: MAC_serveur | MAC dst: MAC_du_PC
"Confirmé ! IP + masque + passerelle + DNS"
Renouvellement automatique : Quand le bail atteint la moitié de sa durée → client envoie DHCPREQUEST → serveur répond DHCPACK.
Agent Relais : L'agent DHCP est R3 (G0/1). Le PC échange avec R3, pas directement avec R5. L'IP src des OFFER/ACK est l'IP de l'interface de R3. R3 relaie en unicast vers R5.

Paramètres fournis par DHCP

  • Adresse IP + durée du bail
  • Masque de sous-réseau
  • Passerelle par défaut (default-router)
  • Serveur DNS (dns-server)
  • @IP du serveur DHCP

Exclusions obligatoires

  • IPs des interfaces routeurs (passerelles)
  • Serveurs avec @IP statique
  • Commande : ip dhcp excluded-address
  • DS : exclure 192.168.10.1 → plage DHCP labo 10 = .10.2 à .10.254

🔀 Agent Relais DHCP

Indispensable quand le serveur DHCP est sur un réseau différent des clients. Le broadcast DISCOVER ne franchit pas les routeurs sans cela.
! Sur l'interface du routeur côté clients (labo)
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip helper-address <@IP_interface_R0>

! Création du pool DHCP sur R0
R0(config)# ip dhcp excluded-address 192.168.10.1
R0(config)# ip dhcp pool Labo_10
R0(dhcp-config)# network 192.168.10.0 255.255.255.0
R0(dhcp-config)# default-router 192.168.10.1
R0(dhcp-config)# dns-server 192.168.100.254

PARTIE IV NAT / PAT — Traduction d'adresses

NAT Statique

1 @privée ↔ 1 @publique fixe. Nb @pub = nb machines. Ne résout pas la pénurie.

NAT Dynamique

Pool d'@pub alloué à la demande. Nb @pub = nb connexions simultanées. Mieux mais limité.

PAT / Overload

N hôtes → 1 seule @publique. Sessions distinguées par numéros de port différents.

⚙️ Configuration NAT / PAT Cisco

! Marquer les interfaces (obligatoire)
R0(config)# interface GigabitEthernet0/1
R0(config-if)# ip nat inside       ← côté réseau privé (labos)
R0(config)# interface GigabitEthernet0/2
R0(config-if)# ip nat outside      ← côté Internet/ISP

! NAT dynamique avec pool d'@IP
R0(config)# ip nat pool PUBLIC 200.60.3.2 200.60.3.7 netmask 255.255.255.0
R0(config)# access-list 1 permit 192.168.0.0 0.0.255.255
R0(config)# ip nat inside source list 1 pool PUBLIC

! PAT (overload) — 1 seule @IP publique pour tous
R0(config)# ip nat inside source list 1 interface GigabitEthernet0/2 overload

! Vérification
R0# show ip nat translations    ← table de traduction active
Interfaces G0/2 de R0 et G0/3 de R5 : Configuration manuelle (pas DHCP) car ce sont des @IP publiques fournies par les ISP (Tunisie Télécom et Orange).

Erreur DS — Routeur Dallas : Interface S0/0/1 configurée avec IP 192.168.111.1 au lieu de 192.168.11.1/30. Le 3e octet est erroné (111 vs 11) → l'interface n'est pas dans le bon sous-réseau → impossible de joindre NYC depuis B.
Pourquoi sécuriser le mode privilégié ? Ce mode donne accès COMPLET à la configuration. Sans mot de passe, n'importe qui peut modifier les routes, supprimer la config, créer des backdoors. Utiliser enable secret (MD5) plutôt que enable password.

PARTIE VI Commutation — Switch et Sécurité des ports

Switch — Rappels couche 2

  • Équipement de couche 2 — adresses MAC
  • Segmente : 1 domaine de collision par port
  • Maintient une table MAC (table CAM)
  • Transmet des trames Ethernet basées sur MAC de destination

CDP — Cisco Discovery Protocol

  • Protocole propriétaire couche 2
  • Détecte les périphériques Cisco voisins
  • Révèle modèle + version IOS → risque sécurité
  • Désactiver si non utilisé : no cdp run

PARTIE VII VLANs et Trunks

📁 4 Types de VLANs

VLAN de données

Trafic utilisateur normal. Réseau IP dédié par VLAN.

VLAN par défaut (VLAN 1)

Tous les ports y appartiennent par défaut. Cisco recommande de ne pas l'utiliser pour la sécurité.

VLAN natif

Trafic non étiqueté sur un trunk. Par défaut = VLAN 1. Configurable.

VLAN de gestion

Dédié à la gestion du switch (SSH, Telnet, SNMP). Interface SVI avec @IP admin.

🔌 Port d'accès (Access)

  • Appartient à un seul VLAN
  • Connecté aux hôtes finaux (PC, imprimantes)
  • Trame non étiquetée côté hôte
  • Le switch ajoute/retire le tag 802.1Q

🔗 Port Trunk (Agrégation)

  • Transporte plusieurs VLANs
  • Entre switches ou switch ↔ routeur
  • Protocole IEEE 802.1Q : tag 4 octets
  • VLAN natif non étiqueté par convention

✅ Avantages des VLANs

AvantageDescription
SécuritéIsolation des groupes
PerformanceRéduction des domaines de diffusion
EfficacitéMeilleures performances réseau

PARTIE VIII Routage Inter-VLAN (Router-on-a-stick)

📊 3 méthodes de routage inter-VLAN

MéthodeDescriptionAvantage / Inconvénient
Routeur classique1 interface physique par VLANSimple mais coûteux en interfaces
Router-on-a-stick ← fréquent examSous-interfaces dot1Q sur 1 lien trunkÉconomique, 1 seul câble physique
Switch Layer 3 (SVI)Switch Virtual Interfaces sur switch L3Haute performance, pas de routeur externe
💡 Concept : Une liaison agrégée 802.1Q transporte plusieurs VLANs via une seule interface physique. Chaque sous-interface = passerelle par défaut d'un VLAN. 
! Étape 1 : activer l'interface physique SANS @IP
R1(config)# interface g0/1
R1(config-if)# no shutdown
R1(config-if)# exit

! Étape 2 : créer sous-interface pour VLAN 10
R1(config)# interface g0/1.10              ← notation : interface.numéro
R1(config-subif)# encapsulation dot1Q 10  ← lie la sous-interface au VLAN 10
R1(config-subif)# ip address 192.168.10.1 255.255.255.0
R1(config-subif)# exit

! Étape 3 : idem pour VLAN 20
R1(config)# interface g0/1.20
R1(config-subif)# encapsulation dot1Q 20
R1(config-subif)# ip address 192.168.20.1 255.255.255.0
Points clés : La notation g0/1.10 = sous-interface 10 de g0/1. L'interface physique principale doit être active (no shutdown) mais sans @IP. Chaque sous-interface = passerelle par défaut pour son VLAN.

PARTIE IX Sécurité Réseau

🟢 SSH (port 22) — À UTILISER

  • Chiffre toutes les données (session complète)
  • Mot de passe non visible sur le réseau
  • Authentification par clé publique possible
  • Standard pour administration réseau
  • Nécessite IOS avec fonctions de chiffrement

🔴 Telnet (port 23) — À ÉVITER

  • Données et mots de passe en clair
  • Facilement interceptable (sniffing)
  • Aucune authentification forte
  • Obsolète et dangereux
  • Encore présent sur de vieux équipements

🔑 Configuration SSH complète sur routeur Cisco

! Prérequis : hostname et domaine OBLIGATOIRES pour la clé RSA
Router(config)# hostname MonRouteur
MonRouteur(config)# ip domain-name monfai.tn

! Générer la clé RSA (minimum 1024 bits recommandé)
MonRouteur(config)# crypto key generate rsa modulus 1024
MonRouteur(config)# ip ssh version 2

! Créer un utilisateur local
MonRouteur(config)# username admin secret monMotDePasse

! Configurer les lignes VTY pour SSH uniquement
MonRouteur(config)# line vty 0 4
MonRouteur(config-line)# transport input ssh   ← bloque Telnet
MonRouteur(config-line)# login local           ← utilise base locale
